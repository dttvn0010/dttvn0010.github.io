Cài đặt môi trường

Cài đặt các phần mềm:

  • JDK 11 (https://jdk.java.net/java-se-ri/11)
  • Maven (https://maven.apache.org/download.cgi)

Để cài đặt, download các phần mềm trên, giải nén ra các thư mục, sau đó thêm các đường dẫn của các thư mục bin (jdk/…/bin), (apache-maven/.../bin) vào đường dẫn (PATH) của hệ thống. (Trên windows: click chuột phải vào My Computer → Advanced System Settings → Environment Variables → Path, sau đó dùng New để thêm các đường dẫn cho các thư mục bin của jdk và maven)

Tạo mới ứng dụng

  • Để tạo mới một ứng dụng Spring boot, có thể sử dụng công cụ https://start.spring.io/ theo hướng dẫn https://spring.io/guides/gs/spring-boot/

  • Ngoài cách trên, để tạo nhanh một ứng dụng, có thể theo các bước sau:
  • Tạo một folder cho ứng dụng, ví dụ sample_project

Xây dựng restful webservice:

  • Các loại restful service và mục đích sử dụng chính của mỗi loại:
  • GET: lấy dữ liệu
  • POST: tạo mới bản ghi
  • PUT: cập nhật bản ghi
  • DELETE: Xóa bản ghi

  • Trong spring boot, các restful service được tạo ra từ các RestController class. Các class này có khai báo @RestController ở trước khai báo class:        

                

@RestController

public class SampleController {

        ...

}

  • Khi có nhiều Controller, để tách biệt các restful service giữa các controller, thường mỗi controller sẽ được gắn một đường dẫn (route) riêng, bằng cách khai báo @RequestMapping ở trước khai báo class:

@RestController

@RequestMapping("/app1")

public class SampleController {

        ...

}

Khi đó, tất cả các restful service do controller cung cấp sẽ được gắn phần đường dẫn của controller vào phía trước đường dẫn của service.

Ví dụ:

package demo;

import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/app1")
public class SampleController {

   @GetMapping("/service1")
   public String service1() {
       return "Hello";
   }
}

Để truy cập đến restful service do hàm service1 cung cấp, cần dùng địa chỉ : http://localhost:8080/app1/service1

           this.studentNo = studentNo;
           this.name = name;        
       }
   }
   
   @SuppressWarnings("serial")

       return student;
   }
   
   @PutMapping("/update_student/{id}")
   public Student updateStudent(@PathVariable("id") int id, @RequestBody Student student) {
       student.id = id;
       for(int i = 0; i < students.size(); i++) {

           }
       }
       return student;
   }

package demo;

import java.util.List;
import java.util.ArrayList;

import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/app1")
public class SampleController {
   
   static class Student {

       public int id;
       public String studentNo;
       public String name;
       
       public Student(int id, String studentNo, String name) {
           this.id = id;
           this.studentNo = studentNo;
           this.name = name;        
       }
   }
   
   @SuppressWarnings("serial")
   List<Student> students = new ArrayList<>() {{
       add(new Student(1, "10001", "Nguyen Van A"));
       add(new Student(2, "10002", "Nguyen Van B"));
   }};
 
   @GetMapping("/get_all_students")
   public List<Student> getAllStudents() {
       return students;
   }
   
   @PostMapping("/add_student")
   public Student addStudent(@RequestBody Student student) {
       var maxId = students.size() > 0? students.get(students.size() - 1).id : 0;
       student.id = maxId + 1;
       students.add(student);
       return student;
   }
   
   @PutMapping("/update_student/{id}")
   public Student updateStudent(@PathVariable("id") int id, @RequestBody Student student) {
       student.id = id;
       for(int i = 0; i < students.size(); i++) {
           if(student.id == students.get(i).id) {
               students.set(i, student);
           }
       }
       return student;
   }
   
   @DeleteMapping("/delete_student/{id}")
   public void deleteStudent(@PathVariable("id") int id) {
       var stOpt = students.stream().filter(st -> st.id == id).findFirst();
       stOpt.ifPresent(st -> students.remove(st));        
   }
}

Cấu trúc của thư mục project

Ý nghĩa của các file và thư mục:

  • pom.xml : File build của maven
  • application.properties: File chứa cấu hình ứng dụng
  • controller: Thư mục chứa các controller class, cung cấp các hàm xử lý cho các restful webservice
  • model: Thư mục chứa các model để mapping với các bảng trong database
  • repository: Thư mục chứa các repository class, dùng để thực hiện các hàm CRUD (Create, Retrieve, Update, Delete) đến database
  • service: Thư mục chứa các service class, dùng để tạo các hàm xử lý business của ứng dụng.

  • Tạo nội dung cho file pom.xml:

So với file pom.xml ở ví dụ trước, file pom.xml ở phần này có thêm các dependency sau:

  • spring-boot-starter-data-jpa: cung cấp các thư viện để truy xuất đến CSDL theo chuẩn JPA (Java persistence API)
  • mysql-connector-java: cung cấp thư viện để kết nối tới database engine MySQL. Nếu ứng dụng sử dụng database engine khác thì phải thêm dependency tương ứng trong file pom.xml.

  • Thiết lập cấu hình database trong file application.properties:

Thêm vào file application.properties các thông tin cấu hình sau:

spring.jpa.hibernate.ddl-auto=update

spring.datasource.url=jdbc:mysql://3.15.210.203/test

spring.datasource.username=admin

spring.datasource.password=abc@123

Các thông tin cấu hình này gồm ip của server database, tên database, username/password để truy nhập database.

  • Tạo model class trong thư mục model:

Trong thư mục src/main/java/demo/model, tạo file Student.java với nội dung dưới đây để thực hiện mapping với bảng student trong database:

package demo.model;

import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.Id;
import javax.persistence.Table;

@Entity
@Table(name = "student")
public class Student {

   @Id 
   @GeneratedValue
   public int id;
   
   public String studentNo;
 
   public String name;

    public String address;
   
}

Ý nghĩa của các annotation:

  • Entity: Class đóng vai trò như một mapping tới một Entity trong database
  • Table: Thực hiện mapping sang bảng có tên tương ứng trong database
  • Id: Trường primary key
  • GeneratedValue: Giá trị của trường này sẽ được springboot tự động sinh ra

  • Tạo file repository:

Trong thư mục src/main/java/demo/repository, tạo file StudentRepository.java với nội dung dưới đây:

package demo.repository;

import org.springframework.data.jpa.repository.JpaRepository;
import demo.model.Student;

public interface StudentRepository extends JpaRepository<Student, Integer> {

}

StudentRepository được khai báo dưới dạng một interface và kế thừa JpaRepository của springboot. Do kế thừa interface này, StudentRepository được cung cấp sẵn các hàm CRUD để truy xuất đến bảng student trong database, một số hàm hay sử dụng bao gồm:

  • findAll() : lấy về toàn bộ danh sách các bản ghi trong database
  • count(): đếm tổng số bản ghi trong database
  • findById(id): lấy về bản ghi có primary key bằng id
  • deleteById(id): xóa bản ghi có primary key bằng id
  • delete(student): xóa bản ghi tương ứng với đối tượng student thuộc class Student
  • save(student): lưu đối tượng student thuộc class Student vào bảng tương ứng trong database. Nếu đối tượng student có id bằng null, lệnh save sẽ tạo mới bản ghi và giá trị id mới tạo ra sẽ được cập nhật vào trường id của student. Nếu đối tượng student đã có id, lệnh save sẽ thực hiện update bản ghi.

Ngoài ra trong StudentRepository  còn cho phép khai báo các hàm tìm kiếm theo các mẫu sau:

  • findBy<Field>() : tìm kiếm theo trường <field>
  • findBy<Field1>And<Field2>() : tìm kiếm theo trường <field1> và <field2>
  • findBy<Field1>And<Field2>And<Field3>() : tìm kiếm theo trường <field1> , <field2> và <field3>

        …

Ví dụ, có thể thêm vào các hàm tìm kiếm như dưới đây trong file StudentRepository.java:

package demo.repository;

import java.util.List;
import java.util.Optional;

import org.springframework.data.jpa.repository.JpaRepository;
import demo.model.Student;

public interface StudentRepository extends JpaRepository<Student, Integer> {

   Optional<Student> findByStudentNo(String studentNo);
   List<Student> findByAddress(String address);
   List<Student> findByNameAndAddress(String name, String address);
}

Lưu ý là chỉ cần khai báo nguyên mẫu hàm, còn nội dung các hàm sẽ được springboot tự động sinh ra qua cơ chế proxy.

File service chứa các hàm để xử lý business của ứng dụng. File này thường gọi các hàm của file repository.

import java.util.Optional;

import org.springframework.stereotype.Service;

import demo.model.Student;

@Service
public class StudentService {

   }

   }

Các service class đều được gắn annotation @Service trước khai báo class, việc này giúp spring khởi tạo các đối tượng service một cách tự động (theo cơ chế của Spring bean). Bên trong service class , đối tượng repository được khai báo theo cách sử dụng annotation @Autowired:

@Autowired private StudentRepository studentRepository;

Đây là cách khởi tạo đối tượng thường được sử dụng của spring.

  • Tạo model class trong thư mục model:

Trong thư mục src/main/java/demo/model, tạo file Student.java với nội dung dưới đây để thực hiện mapping với bảng student trong database:

package demo.model;

import org.bson.types.ObjectId;
import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection="student")
public class Student {
   
   static class Address {
       public String street;
       public String city;
   }

   @Id 
   public ObjectId id;
   
   public String studentNo;
   
   public String name;
   
   public Address address;
   
   public String getId() {
       return id != null? id.toHexString(): null;
   }
}

Ở đây, việc mapping giữa java class với bảng trong mongodb được thực hiện bởi annotation @Document thay cho @Table như với SQL ở ví dụ trước. Ngoài ra các trong các entity class có thể chứa các đối tượng thuộc các class con khác (class Address ở ví dụ trên) để phù hợp với mô hình dữ liệu kiểu nested của mongodb.

Ngoài ra, đối tượng của mongodb sử dụng kiểu dữ liệu ObjectId (có thể chuyển đổi thành một hex string) cho trường id, thay cho kiểu int/long như ở SQL.
